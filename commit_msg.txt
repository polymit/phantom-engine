feat(core): implement top-level browser error hierarchy

The workspace previously lacked a unified, typed error handling system, 
relying on scattered tuples or string-based errors across crates.

This change introduces the centralized `BrowserError` hierarchy using 
the `thiserror` crate. It securely wraps domain-specific variants:
- NetworkError (DNS, TLS, HTTP, timeouts)
- DomError (selectors, interactability, stale elements)
- JsError (uncaught exceptions, timeouts, OOM)
- NavigationError (loads, timeouts, redirects)
- BrowserSessionError (budgets, expirations, missing tabs)
- InternalError (panics, runtime pools)

Additional standard constants for engine limits (JS execution timeouts, 
heap sizes for QuickJS/V8, and session limits) are defined. `From` 
implementations are provided to convert string-based JavaScript errors 
directly into the typed `JsError` structure.

Test expectations in the core pipeline suite have also been aligned 
with recent layout computation metrics.
